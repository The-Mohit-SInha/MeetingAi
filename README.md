# AI Meeting-to-Action Management System

A modern, production-ready meeting management and action item tracking application built with React, TypeScript, Tailwind CSS, and Supabase.

## ✨ Features

- **Meeting Management**: Create, view, edit, and delete meetings with full details
- **Action Items**: Track tasks and action items from meetings
- **Participants**: Manage meeting participants and attendees
- **Analytics**: View statistics and insights on meetings and action items
- **Notifications**: Real-time notifications for important events
- **Calendar View**: Visualize meetings in a calendar format
- **Profile Management**: Update user profile and avatar
- **Settings**: Customize theme, compact mode, and notification preferences
- **Data Export**: Export meetings and action items to CSV or JSON

## 🛠️ Tech Stack

- **Frontend**: React 18 with TypeScript
- **Routing**: React Router v7
- **Styling**: Tailwind CSS v4 with custom design system
- **Animations**: Motion (Framer Motion)
- **Icons**: Lucide React
- **Charts**: Recharts
- **Backend**: Supabase (PostgreSQL + Auth + Storage)
- **Build Tool**: Vite

## 🚀 Getting Started

### Prerequisites

- Node.js 18+ and npm/pnpm
- Supabase account and project

### Installation

1. Clone the repository
2. Install dependencies:
   ```bash
   npm install
   # or
   pnpm install
   ```

3. Set up Supabase:
   - Create a Supabase project at https://supabase.com
   - Update `/utils/supabase/info.tsx` with your project credentials (auto-generated by Figma Make)
   - Run the database migration from `/supabase/migrations/20260402000000_initial_schema.sql` in the Supabase SQL Editor

4. Start the development server:
   ```bash
   npm run dev
   # or
   pnpm dev
   ```

5. Open http://localhost:5173 in your browser

- Indexed queries for performance
- Avatar storage in Supabase Storage

## 🔧 Configuration

### Compact Mode
Toggle between compact and larger UI in Settings. Default is OFF (larger design).

### Theme
Light and dark modes available. Default is light mode.

### Notifications
Configure email, push, and Slack notifications in Settings.

## 📊 Database Schema

Tables:
- `users` - User profiles
- `meetings` - Meeting records
- `meeting_participants` - Participants for each meeting
- `action_items` - Tasks and action items
- `notifications` - User notifications
- `user_settings` - User preferences

The application is production-ready and can be deployed to:
